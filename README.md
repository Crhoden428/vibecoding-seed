# Vibecoding Seed

One file. Drop it in. Claude does the rest.

---

## Recommended: VS Code + Claude Code

For the full system, use **VS Code with the Claude Code extension**.
This is the setup this system is designed for.

What you get that the web version cannot do:
- Auto-approve — Claude works without stopping to ask permission
- Live status bar — see exactly what Claude is doing in real-time
- Lint on every edit — bad code caught the moment it's written
- Quality gate — tests must pass before Claude declares done
- Force push blocked — destructive git ops stopped before they run

Download VS Code: https://code.visualstudio.com
Install Claude Code: open VS Code → Extensions → search "Claude Code"

---

## How to use it

**VS Code Claude Code (recommended):**
1. Download `vibecoding-setup.md` from this repo
2. Drop it into your project folder
3. Say: **"Read vibecoding-setup.md and carry out the bootstrap instructions."**

**Claude.ai Projects (web, limited):**
1. Download `vibecoding-setup.md` from this repo
2. Upload it to your Claude Project (Add content → upload file)
3. Say: **"Read vibecoding-setup.md and carry out the bootstrap instructions."**

Note: the web version sets up working rules and project memory only.
Hooks, quality gates, and auto-approve require VS Code Claude Code.

Works for new projects and existing ones. Claude detects the situation and handles it.

---

## What gets set up (VS Code Claude Code)

- Auto-approve — Claude works without asking permission for every action
- Live status bar in VS Code showing branch and current task
- Lint on every file edit — bad code caught immediately
- Quality gate before done — tests must pass before Claude declares victory
- Force push blocked — destructive git ops stopped automatically
- Multiagent by default — parallel research, coding, and testing
- Persistent project memory — context loaded every session automatically
